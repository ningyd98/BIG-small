# 中文代码注释与说明规范

本项目的代码注释以“帮助维护者理解业务约束和安全边界”为目标，不追求逐行翻译代码。自解释的赋值、简单返回值和框架样板不需要注释；涉及安全、并发、状态机、持久化、硬件边界、证据生成和前后端契约的代码必须有中文说明。

## 必须说明的位置

- 模块顶部：说明模块职责、输入输出边界，以及是否允许接触真实硬件。
- 公共类和公共函数：说明调用方应该依赖的语义，而不是复述参数名称。
- 状态机、CAS、租约、恢复、取消、超时：说明为什么采用该流程，以及失败时如何 fail-closed。
- Artifact、证据、日志、导出：说明脱敏、路径、可复现性和验收关系。
- 前端服务和组件：说明数据来源、展示含义、空状态、降级和权限边界。
- Verifier 与测试：说明覆盖的验收点，避免后续维护者误把 Mock 结果当作真实验收。

## 不应添加的注释

- 不写“给变量赋值”“返回结果”这类机械注释。
- 不写与代码不一致的愿望式说明。
- 不用注释隐藏技术债；技术债应写入文档或 issue 风格的明确说明。
- 不在注释中记录 token、IP、serial、用户名、本机绝对路径或 credential。

## 审计口径

`scripts/check_chinese_comments.py` 用于检查重点代码区域是否具有中文模块说明。它是底线审计，不代表完整质量评审；通过脚本后仍需要人工判断注释是否解释了业务和安全语义。
